# Dungeon-challenge

Проект представляет собой выполнение тестового задания, то есть симуляцию прохождения пользователем подземелья.

Весь проект выполнен на языке Go, также созданы unit тесты, Makefile.

## Запуск
При помощи Makefile можно легко запустить проект, тесты, линтер, то есть все самое необходимое:
#### Запуск проекта
```shell
make run
```
#### Запуск тестов
```shell
make test
```
#### Запуск тестов с покрытием
```shell
make cover
```
#### Запуск линтера
```shell
make lint
```

## Сложности
Задание полностью не покрывало всю бизнес-логику проекта, поэтому какие-то моменты приходилось исполнять исходя из логики.
- Переход на следующий уровень и на предыдущий
  - В задании четко ничего не сказано по этим пунктам. Я решил, что на следующий этаж пользователь может перейти только в том случае, когда зачищен весь текущий этаж. Назад же спуститься может в любой момент времени, но если находится не ниже 1 этажа.
- Формат output
  - В задании (README.md):
    В таблице указан следующий текст для команды с ID = 31:

    Player [`id`] disqualified

    Ниже в примере написано следующее:

    [14:10:00] Player [3] is disqualified

    То есть уже есть 'is' - в задании используется формат:

    Player [`id`] disqualified - так как он находится не просто в примере, а в таблице с описанием входящих и исходящих событий.
